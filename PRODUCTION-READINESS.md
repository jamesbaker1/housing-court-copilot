# Production-Readiness Review — Housing Court Copilot

_Review date: 2026-06-22. Scope: software production-readiness with an emphasis on the experience for the actual users — low-income, frequently limited-English, low-digital-literacy, scared tenants on a borrowed phone with ~10 days. Read-only review; findings below are adversarially verified (refuted findings dropped, severities re-calibrated). Source of truth: `REVIEW-PLAYBOOK.md` §1–2 and the dimension findings + adversarial verdicts feeding this doc._

---

## 1. Overall verdict

**The software is NOT production-ready.** It is well-architected and the four safety invariants and boundary auth verifiably hold today (no regression from the prior `REVIEW.md` fixes) — but the live application is **functionally broken past step 3 for every real user in its intended production configuration.** The four post-intake LLM surfaces (chat, defenses, answer draft, stipulation reviewer) call `verifyTurnstile()` server-side and fail closed (403) whenever `TURNSTILE_SECRET_KEY` is set — which the deploy docs *require* — yet none of those client callers ever render a Turnstile widget or send a token. So "configured correctly per the runbook" = "the entire Q&A/informational core is dead," surfaced only as a generic English error with no recovery path. On top of that single blocker, the project ships with **zero automated tests and no CI** (nothing guards the safety invariants against regression on a public, contribution-inviting repo), **no health check / alerting / staging tier** (silent broken deploys are undetectable), a **documented-but-unwired PII-at-rest encryption control** (Cases persist as plaintext JSON in D1), and significant **accessibility and i18n gaps** for exactly the vulnerable population the product targets. None of the proposed fixes weaken the four safety invariants.

**Phase-0 gates remain a separate, non-software prerequisite.** Even once the software work below lands, real-tenant use is still gated on the two Phase-0 prerequisites that are out of scope here: (a) a NY-licensed attorney must populate and validate the deadline + eligibility configuration (the engine currently runs on `UNVALIDATED_DEADLINE_CONFIG`), and (b) a funded legal-aid provider partner must back the human handoff (including the verified hotline/Right-to-Counsel intake number, which is currently an unverified `311` placeholder). Production-ready software is necessary but **not** sufficient.
